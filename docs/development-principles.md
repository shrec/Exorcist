# Exorcist Development Principles

This file now serves as a compact principle index. Detailed architecture and
plugin rules live in the canonical working documents.

## Principles

1. Keep the shell thin.
2. Prefer shared reuse over copy-paste.
3. Let plugins own feature workflows.
4. Let profiles assemble capability packs.
5. Stabilize contracts before adding more surface area.
6. Treat bundled and external plugins by the same rules.
7. Reorganize first when the system is already functionally rich.
